## 1. Firebase 프로젝트 설정 (1회, 약 10분)
1. https://console.firebase.google.com → 프로젝트 만들기
2. **빌드 > Realtime Database** → 데이터베이스 만들기 (지역: asia-southeast1 권장) → 규칙 탭에 아래 입력:
```json
{ "rules": { ".read": "auth != null", ".write": "auth != null" } }
```
3. **빌드 > Authentication** → 시작하기 → 로그인 방법에서 **익명** 사용 설정
4. **빌드 > Storage** → 시작하기 (이미지 일괄 업로드용) → 규칙도 auth != null 로
5. **프로젝트 설정(톱니) > 일반 > 내 앱 > 웹 앱 추가(</>)** → 표시되는 `firebaseConfig` 객체 복사

## 2. 설정 붙여넣기
세 파일 모두 상단의 `FIREBASE_CONFIG = { ... }` 안에 복사한 값을 붙여넣습니다.
- 참가자 앱: 설정이 비어 있으면 **로컬 데모 모드**(봇배틀·데모순위)로 작동 → 넣는 순간 온라인 전환
- 관리자 콘솔: `ADMIN_PASS = "hwansu2026"` 을 원하는 암호로 변경

## 3. 배포 (기존 방식 그대로)
GitHub Pages 저장소에 3개 파일 업로드 → 각 URL 접속.
- 참가자용 URL은 QR로 만들어 입장 안내 슬라이드에 게시
- 걸음 센서는 HTTPS 필수 (GitHub Pages = 자동 충족)

## 4. 행사 당일 운영 순서
1. **[관리자]** 문제은행: "기본 60문항 업로드" 클릭 (또는 직무역량 문항 JSON 일괄 업로드)
2. **[스크린]** 프로젝터로 열기 → 관리자에서 "리더보드" 모드
3. 참가자 입장 → 닉네임 등록 (관리자 모니터링에 실시간 표시)
4. **[관리자]** "1라운드" + 타이머 30분 → 참가자 앱 탐험 활성화
5. 1R 종료 → **"2라운드"** + 타이머 30분 → 배틀·치유 활성화, **"샘 열기"** + 스크린 "치유의 샘 코드" 모드(부스 옆 보조 모니터 권장, 메인은 리더보드 유지 가능 — 모드는 언제든 전환)
6. 수시로 공지 발송 ("5분 남았습니다!" 등) → 전원 피드에 표시
7. **"게임 종료"** → 전원 앱이 잠기고 **복습 노트 자동 표시**
8. 리더보드 모드로 시상!

## 5. 실전 배틀 방법 (참가자 안내용)
- 두 사람이 만나서 → 한쪽이 "배틀 코드 만들기" → 4자리 코드를 상대에게 보여줌 → 상대가 "코드 입력해 참가"
- 각자 환수 선택(상대 비공개) → 스킬 3개 비밀 제출 → 동시 공개!
- 승수·HP·점수는 자동 저장 (새로고침해도 유지)

## 6. Midjourney 작화 교체 (준비되면)

## 7. 알아둘 제한사항 (정직 노트)
- 서버 함수 없이 RTDB만 쓰는 구조라, 치유 코드·블라인드 픽은 **UI 차원에서 보장**됩니다. 개발자 도구로 DB를 직접 열람하는 수준의 부정행위까지 막지는 못하지만, 사내 행사 규모에선 현실적 위험이 낮고 기존 프로젝트들과 동일한 신뢰 모델입니다.
